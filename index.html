WAV is used for audio files. It is stored in binary format and is organized into chunks. It contains a format chunk and a data chunk. The format chunk contains the sample rate and width, and the data chunk contains the samples of the audio. It also has the possibility of containing another chunk that contains items such as copyright information and composition credits. It is most commonly used on PC and for CD audio. It works by taking soundwaves and converting them into a digital, binary format. It was created in collaboration with Microsoft and IBM for Windows95 (Webopedia), released in August of 1995. Similar audio formats around during the time were MP3 format and the RIFF format. This format is open source, and therefore can be used, accessed, and created on multiple platforms by anyone who wishes to do so. This type of file is still in use and common. WAV does not use compression, and therefore takes up a lot of space. It is also lossless, (Gordon) and therefore sound is conserved. The typical file size is anywhere from 50.4 MB to 329 MB (DSD). The size is dependent upon the sampling frequency of the audio. Similar formats to WAV are FLAC, DSF, and MP3. The advantage that WAV presents over DSF and MP3 is that WAV is lossless, whereas these are lossy. FLAC, however, utilizes compression and therefore has a much smaller file size than WAV (Gordon). The large file size of WAV is its main disadvantage. However, it is a very universal format that can be accessed on many devices, and is good for those wishing to edit the audio contained in the file (Gordon). AAC is an audio file format,and is the standard audio format used by Apple (Gordon). It is unknown to the general public exactly how this file is organized and functions. It is the audio file type used by all Apple products, alongside Nintendo products. AAC, or Advanced Audio Coding, was developed by AT&T Bell Laboratories, Fraunhofer IIS, Dolby Laboratories, Sony Corporation and Nokia. It was first introduced in 1997, but has been edited and re-released multiple times since its initial release. Similar formats of the time were MP3, WAV, and FLAC. The typical file size is around 30-40 MB, but, as with all audio files, depends on the length and frequency of the file. Because it is the standard audio file of both Apple and Nintendo, it is still highly in use. AAC uses compression (FileInfo), and therefore typically has a file size smaller than some audio formats. It is lossy, and therefore some of the sound quality is lost through compression. However, it is less lossy than most lossy formats, making it highly efficient (Gordon). A typical file size before compression can be up to 500 MB, therefore it compresses files by up to 94 percent. The most similar format to AAC is MP3. However, AAC is a better choice because it has a much smaller file size with the same quality. The easy accessibility of conversion to AAC files through iTunes makes it easily accessible, one of its strongest points.

So Which Is Better?

Now that you understand the formats, it is time to figure out once and for all whether WAV or AAC is the superior audio file format. In my opinion, AAC is the superior format. Although it is lossy, the use of compression that creates a smaller file size makes up for it. Through the use of AAC, you could have, for example, fivesongs that take up 100 MB total, but with WAV, youmay onlybe ableto have one with that amount of data. AAC is also a better choice than WAV because it is increasingly becoming the most widely recognized audio format, as it is used by Apple. As Apple continues to dominate the technological world, its formats will continue to become superior and more widely used than others. Lastly, it is the better option because it requires liscensing. This means that your files will be protected from copyright infringement and other thefts. Therefore, AAC is a better choice of audio formatting over WAV.
